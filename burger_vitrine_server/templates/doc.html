Versions

Direct addressing

In the simplest scenario a resource is addressed by a single version as they appear on /list

Examples:

  • 1.2.3
  • b1.2

Relative addressing

Versions can also be addressed by type and relative position with git-like expressions

Examples:

  • 1.2.3
  • latest
  • release
  • snapshot~3
  • prerelease^

Version Diff

Resources addressed with the versions keyword accept a single version or two of them joined by three periods (using an ellipsis character results in a redirect).

Examples:

  • 1.2.3...1.4.5
  • latest^...latest

Notes

Alternative JSON addresses

JSON resources are also available on /versions.json or by sending an application/json accept header to /versions
